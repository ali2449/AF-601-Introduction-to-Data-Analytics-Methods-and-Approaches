Section 1: Variables and Assignment

Variables are used to store data in Python. A variable is a named location in memory that holds a value, enhancing code reusability by allowing values to be referenced by name. Assignment is performed using the = operator. For example, in a business setting, a variable might store sales figures.

Section 2: Data Types

Data types specify the nature of stored values. Common types include integers (int) for whole numbers, floats for decimal values, strings (str) for text, and booleans (bool) for true/false conditions. Understanding data types is crucial because operations are type-dependent, and Python automatically infers types.

In [2]:
# Examples of data types
age = 30  # int
height = 5.9  # float

is_student = True  # bool

Section 1: Conditional Statements - If/Else

The if statement evaluates conditions to execute code blocks selectively, enabling decision-making logic. Use if/else for alternative paths. Correct indentation (4 spaces) is crucial.

In [1]:
# If/else example
sales = 1000
if sales > 800:
    print("Good sales")
else:
    print("Low sales")
Good sales

Section 2: For Loops

for loops iterate over sequences (like lists or ranges), executing code for each item. They are ideal for processing a known number of items.

In [2]:
# For loop example
patients = [1, 2, 3]
for patient in patients:
    print(f"Process patient {patient}")
Process patient 1 Process patient 2 Process patient 3

Section 3: While Loops

while loops execute as long as a condition remains true. They are suitable when the number of iterations is unknown beforehand. Ensure the condition eventually becomes false to avoid infinite loops.

In [3]:
# While loop example
balance = 100
while balance > 0:
    balance -= 20
    print(balance)
80 60 40 20 0

Section 4: Nesting and Control Statements

Nesting combines loops and conditionals for complex logic. break terminates a loop, while continue skips to the next iteration.

In [4]:
# Nested example
for i in range(3):
    if i == 1:
        continue
    print(i)
0 2

From previous modules, you have mastered variables, data types, control flow, and user input. These concepts enable the organization of code into reusable functions for more efficient and readable programs.

Section 1: Defining and Calling Functions

Functions are named blocks of code that perform specific tasks. You define a function using the def keyword and call it by its name followed by parentheses.

In [1]:
# Define a simple function
def calculate_profit():
    sales = 1000
    costs = 800
    print(sales - costs)

# Call the function

200

Section 2: Parameters and Arguments

Parameters allow functions to accept inputs (arguments), making them more flexible and dynamic. You define parameters inside the parentheses in the function definition.

In [2]:
# Function with parameters
def check_age(age):
    if age > 18:
        print("Adult")

check_age(25)
Adult

Section 3: Return Values

The return statement sends a value back from the function to the caller. This allows the result of a function to be stored in a variable or used in other operations.

In [3]:
# Function with a return value
def get_profit(sales, costs):
    return sales - costs

profit = get_profit(1000, 800)
print(profit)
200

Error handling with try...except blocks manages exceptions gracefully, preventing your program from crashing due to unexpected issues, like invalid user input.

Comments (#) and docstrings ("""...""") are crucial for documenting your code, making it easier for you and others to understand and maintain.

Module 4: Lists & Tuples for Data Organization

From previous modules, you have learned that functions organize code, parameters and return values enhance flexibility, error handling ensures robustness, and comments improve clarity. These concepts support working with collections of data using lists and tuples.

Chapter 1: Comprehensive Tutorial

Business Context

Lists and tuples are structures for storing multiple items. Businesses utilize lists for collections like sales figures and tuples for immutable data like patient vitals. Their importance lies in efficient data organization and time savings in analysis. In informatics, lists can hold gene sequences.

Section 1: Creating Lists

Lists are ordered, mutable (changeable) collections of items, defined using square brackets []. They are one of the most versatile data types in Python.

In [1]:
# Create a list of sales
sales = [100, 200, 300]
print(sales)
[100, 200, 300]

Section 2: List Operations

You can modify lists by adding items with .append() or removing them with .pop(). These operations allow you to dynamically update your collections of data.

In [2]:
# List operations
patients = ["Alice", "Bob"]
patients.append("Charlie")  # Add an item to the end
patients.pop(0)             # Remove the item at index 0
print(patients)
['Bob', 'Charlie']

Section 3: Accessing and Modifying List Elements

Elements in a list are accessed by their index, which starts at 0. You can also change the value of an element by assigning a new value to a specific index.

In [3]:
# Accessing and modifying list elements
genes = ["A", "B", "C"]
print(genes[0])   # Access the first element
genes[1] = "D"      # Change the second element
print(genes)
A ['A', 'D', 'C']

Section 4: Introduction to Tuples

Tuples are ordered, immutable (unchangeable) collections defined with parentheses (). They are useful for data that should not be modified after creation, such as coordinates or fixed patient IDs.

In [4]:
# Create a tuple for latitude and longitude
coords = (35.9, -79.0)
print(coords)
(35.9, -79.0)

Section 5: Immutability and Use Cases

Immutability ensures that a tuple's data cannot be altered, which prevents accidental modifications and maintains data integrity. Attempting to change a tuple will raise a TypeError.

In [5]:
# Immutability example
try:
    coords[0] = 36.0  # This will raise an error
except TypeError:
    print("Cannot change tuple")
Cannot change tuple

Glossary & FAQ

Glossary

  • List: A mutable (changeable), ordered sequence of elements.
  • Tuple: An immutable (unchangeable), ordered sequence of elements.
  • Index: The position of an element in a list or tuple, starting from 0.
  • Append: A method to add an item to the end of a list.
  • Pop: A method to remove an item from a list at a specified index.
  • Mutable: Capable of being changed after creation.
  • Immutable: Incapable of being changed after creation.

FAQ

  • Q: What is the main difference between lists and tuples? A: Lists are mutable (can be changed), while tuples are immutable (cannot be changed).
  • Q: Why would you use a tuple instead of a list? A: Use a tuple when you want to ensure that the data cannot be accidentally modified, which is good for protecting data integrity.
  • Q: How do you access elements in a list or tuple? A: You access elements using their index in square brackets, like my_list[0].

Module 5: Dictionaries & Sets for Data Relationships

From previous modules, you have learned that lists and tuples store sequences of data. These collections lay the foundation for managing more complex data relationships and ensuring the uniqueness of elements using dictionaries and sets.

Chapter 1: Comprehensive Tutorial

Business Context

Dictionaries and sets facilitate handling data relationships and uniqueness. Businesses employ dictionaries for key-value mappings, such as products to prices, and sets for unique items like customer IDs. In healthcare, dictionaries can map patients to details. Their value lies in efficient lookups and ensuring uniqueness, saving time in data queries.

Section 1: Creating Dictionaries

Dictionaries are unordered, mutable collections of key-value pairs, defined with curly braces {}. They allow for fast data retrieval based on a unique key.

In [1]:
# Create a dictionary of a patient
patient = {'name': 'Alice', 'age': 30}
print(patient)
{'name': 'Alice', 'age': 30}

Section 2: Dictionary Operations

You can add, remove, and update key-value pairs in a dictionary. Use square bracket notation to add or update an entry, and the del keyword to remove one.

In [2]:
# Dictionary operations
sales = {'item1': 100, 'item2': 200}
sales['item3'] = 300  # Add a new item
del sales['item1']     # Remove an item
print(sales)
{'item2': 200, 'item3': 300}

Section 3: Accessing and Modifying Dictionary Values

Access and modify values using their corresponding keys. This provides a meaningful way to retrieve and update data, which is more efficient than index-based access for relational data.

In [3]:
# Accessing and modifying dictionary values
genes = {'A': 'trait1', 'B': 'trait2'}
genes['B'] = 'trait3'  # Modify a value
print(genes['A'])      # Access a value
trait1

Section 4: Creating Sets

Sets are unordered, mutable collections of unique items. They are defined using curly braces {} without key-value pairs and automatically eliminate any duplicate entries.

In [4]:
# Create a set of unique genes
unique_genes = {'A', 'B', 'A'}  # Duplicates are automatically removed
print(unique_genes)
{'A', 'B'}

Section 5: Set Operations

Set operations include mathematical tasks like union (|) to combine sets and intersection (&) to find common elements. These are useful for comparing collections of items.

In [5]:
# Set operations
set1 = {'A', 'B'}
set2 = {'B', 'C'}
union = set1 | set2  # Union of set1 and set2
print(union)
{'A', 'C', 'B'}

Glossary & FAQ

Glossary

  • Dictionary: A mutable collection of unique key-value pairs.
  • Set: A mutable collection of unique, unordered elements.
  • Key: The unique identifier for a value in a dictionary.
  • Value: The data associated with a key.
  • Union: An operation that combines two sets into one, including all unique elements from both.
  • Intersection: An operation that finds the common elements between two sets.

FAQ

  • Q: What is the main difference between a dictionary and a list? A: Dictionaries use keys for access, while lists use numerical indexes. Dictionaries are better for relational data.
  • Q: How do sets handle duplicate items? A: They are automatically and silently removed, ensuring every element in a set is unique.
  • Q: What happens if you try to access a key that doesn't exist in a dictionary? A: It will raise a KeyError. You can use the .get() method to avoid this by providing a default value.

Module 6: Modules, Packages & Object-Oriented Basics

From previous modules, you have learned about lists, tuples, dictionaries, and sets for data organization. You can define functions for reusable code and handle errors. These concepts prepare you for structuring larger programs using modules and object-oriented principles.

Chapter 1: Comprehensive Tutorial

Business Context

Modules and packages organize code into files and directories, while object-oriented programming (OOP) models real-world entities. Businesses leverage modules for shared functions, such as importing the math module for calculations. In healthcare, classes can represent patient objects. Their value lies in creating scalable and maintainable software, reducing development time.

Section 1: Importing Modules

Importing allows you to load code from other Python files or libraries, promoting code reuse. The import statement makes the code in one module available in another.

In [1]:
# Importing the math module
import math
print(math.pi)  # Use the pi constant from the math module
3.141592653589793

Section 2: Defining Classes

A class is a blueprint for creating objects. It defines a set of attributes and methods that the created objects will have. The __init__ method is a special method used to initialize an object's attributes.

In [2]:
# Defining a Patient class
class Patient:
    def __init__(self, name, age):
        self.name = name
        self.age = age

# Creating an instance of the Patient class
p = Patient("Alice", 30)
print(p.name)
Alice

Section 3: Methods in Classes

Methods are functions defined inside a class that operate on the object's data (attributes). The first parameter of a method is always self, which refers to the instance of the class.

In [3]:
# A class with a method
class Gene:
    def __init__(self, sequence):
        self.sequence = sequence
    
    def get_length(self):
        return len(self.sequence)

g = Gene("ACTG")
print(g.get_length())
4

Section 4: Attributes and Instance Variables

Attributes are variables associated with an object that store its state. They are typically defined within the __init__ method using self.attribute_name = value.

In [4]:
# A class with attributes
class MapPoint:
    def __init__(self, lat, lon):
        self.lat = lat
        self.lon = lon

point = MapPoint(35.9, -79.0)
print(point.lat)
35.9

Module 7: Advanced Python Techniques

From previous modules, you have learned about modules and packages for code organization, and you can define classes with attributes and methods to model objects. These concepts support the advanced techniques for more concise and efficient coding that we will cover in this module.

Chapter 1: Comprehensive Tutorial

Business Context

Advanced Python techniques enhance code conciseness and efficiency. Businesses employ list comprehensions for data processing, lambda functions for quick definitions, and generators for memory efficiency. Debugging and optimization ensure reliable and fast code. In healthcare, comprehensions can filter patient lists. Their value lies in handling large datasets more effectively and conserving resources.

Section 1: List Comprehensions

List comprehensions offer a shorter, more readable syntax for creating lists. They can mimic for loops and if conditions within a single line.

In [1]:
# List comprehension to create a list of squares
squares = [x**2 for x in range(5)]
print(squares)
[0, 1, 4, 9, 16]

Section 2: Lambda Functions

Lambda functions are small, anonymous functions defined with the lambda keyword. They can take any number of arguments but can only have one expression. They are often used when a simple function is needed for a short period.

In [2]:
# A lambda function that doubles its input
double = lambda x: x * 2
print(double(5))

Section 1: Descriptive Statistics

Descriptive statistics provide simple summaries about the sample and the measures. Measures like mean, median, and standard deviation describe the central tendency and spread of the data.

In [1]:
# Descriptive statistics with Pandas
import pandas as pd
df = pd.DataFrame({'Sales': [100, 150, 200, 160, 140]})
mean_sales = df['Sales'].mean()
median_sales = df['Sales'].median()
std_sales = df['Sales'].std()
print(f"Mean: {mean_sales}, Median: {median_sales}, Std Dev: {std_sales:.2f}")
Mean: 150.0, Median: 150.0, Std Dev: 35.36

Section 2: Probability Distributions

Probability distributions are mathematical functions that describe the likelihood of different possible values for a variable. The normal distribution (bell curve) is a common example.

In [2]:
# Plotting a normal distribution
from scipy.stats import norm
import numpy as np
import matplotlib.pyplot as plt
x = np.linspace(-3, 3, 100)
pdf = norm.pdf(x)
plt.plot(x, pdf)
plt.title('Standard Normal Distribution')
plt.show()
(A plot of the normal distribution bell curve is displayed)

Section 3: Hypothesis Testing

Hypothesis testing is a statistical method used to make decisions or draw conclusions about a population based on sample data. A common example is the t-test, which assesses whether the means of two groups are significantly different.

In [3]:
# Hypothesis testing with a t-test
from scipy.stats import ttest_ind
group1 = [1, 2, 3, 3, 4]
group2 = [4, 5, 6, 5, 7]
t_stat, p_value = ttest_ind(group1, group2)
print(f"P-value: {p_value:.4f}")
P-value: 0.0034

Glossary & FAQ

Glossary

  • Mean/Median: Measures of the central tendency of a dataset.
  • Standard Deviation: A measure of the amount of variation or dispersion of a set of values.
  • Distribution: A function that shows the possible values for a variable and how often they occur.
  • P-value: The probability of observing data as extreme as, or more extreme than, what was observed, assuming the null hypothesis is true.
  • Matplotlib/Seaborn: Python libraries for creating data visualizations.
  • Heatmap: A graphical representation of data where values are depicted by color.

FAQ

  • Q: When should I use the mean vs. the median? A: Use the median when your data has significant outliers, as it is less sensitive to extreme values than the mean.
  • Q: What does a low p-value (e.g., < 0.05) mean? A: It indicates that there is strong evidence against the null hypothesis, so you reject the null hypothesis. It suggests your results are statistically significant.
  • Q: What's the main difference between Matplotlib and Seaborn? A: Matplotlib is a lower-level library that provides more control, while Seaborn is a higher-level library that simplifies the creation of common, attractive statistical plots.

Section 1: Machine Learning Basics

Machine learning algorithms learn patterns from data. A core concept is splitting data into training and testing sets to evaluate how well a model generalizes to new, unseen data. We use the scikit-learn library for this.

In [1]:
# Splitting data into training and testing sets
from sklearn.model_selection import train_test_split
# X represents features, y represents the target label
X = [[0], [1], [2], [3]]
y = [0, 0, 1, 1]
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.5, random_state=42)
print(f"Training Features: {X_train}")
print(f"Testing Features: {X_test}")
Training Features: [[3], [0]] Testing Features: [[2], [1]]

Section 2: Scikit-learn Regression

Regression is a type of supervised learning used to predict a continuous outcome (e.g., a price or temperature). Linear regression is a basic algorithm that finds the best-fitting line through the data.

In [2]:
# Linear Regression example
from sklearn.linear_model import LinearRegression
X = [[1], [2], [3]]
y = [2, 4, 6]
model = LinearRegression().fit(X, y)
prediction = model.predict([[4]])
print(f"Prediction for input 4: {prediction}")
Prediction for input 4: [8.]

Section 3: Scikit-learn Classification

Classification is a type of supervised learning used to predict a categorical label (e.g., 'spam' or 'not spam'). A Decision Tree is an algorithm that learns a set of rules to make predictions.

In [3]:
# Decision Tree Classification example
from sklearn.tree import DecisionTreeClassifier
X = [[0, 0], [1, 1]]
y = [0, 1] # Labels for each class
model = DecisionTreeClassifier().fit(X, y)
prediction = model.predict([[2, 2]])
print(f"Predicted class for [2, 2]: {prediction}")
Predicted class for [2, 2]: [1]

Glossary & FAQ

Glossary

  • Machine Learning: Algorithms that learn patterns from data to make predictions.
  • Supervised Learning: ML where the model learns from labeled data (features and corresponding targets).
  • Regression/Classification: Types of supervised learning for predicting continuous and categorical outcomes, respectively.

FAQ

  • Q: What is the difference between machine learning and statistics? A: While they are closely related, ML often emphasizes predictive accuracy, whereas traditional statistics may focus more on inference and explaining relationships.
  • Q: What is an API key? A: An API key is like a password that grants you access to an API. Many APIs require a key to track usage and for security.
  • Q: Is web scraping legal and ethical? A: It depends. You should always check a website's `robots.txt` file and its terms of service. Scraping public data is generally acceptable, but scraping private or copyrighted data is not.

Section 1: SQL Basics and Integration

SQL (Structured Query Language) is the standard language for managing relational databases. Python's built-in sqlite3 module allows you to work with a lightweight SQL database directly in your code.

In [1]:
# Basic SQL operations with sqlite3
import sqlite3
conn = sqlite3.connect(':memory:')  # Create an in-memory database
cur = conn.cursor()
cur.execute('CREATE TABLE patients (name TEXT, age INTEGER)')
cur.execute("INSERT INTO patients VALUES ('Alice', 30)")
all_patients = cur.execute('SELECT * FROM patients').fetchall()
print(all_patients)
conn.close()
[('Alice', 30)]
