MSIA Program Overview

The Informatics and Analytics (MSIA) program at UNCG is a 30-credit master's degree. It develops leaders and problem-solvers in data science. It has six core courses. These build a strong foundation in data science principles. Students choose one of six concentrations. These are Advanced Data Analytics, Bioinformatics, Computational Analytics, Cultural Analytics, Geospatial Analytics, or Health Analytics. The program is offered in-person and online. It teaches skills in managing, manipulating, and analyzing data. This includes structured and unstructured data. The goal is to generate useful insights. Principles include programming for analytics, data visualization, statistical modeling, database design, and high performance computing. Students come from diverse backgrounds. These include business, science, humanities, healthcare, geography, and more. Some have basic programming and statistics experience. Others may need instructor permission for prerequisites.

Python Setup Guide

To start with Python, install Anaconda. Anaconda bundles Python and useful libraries. It makes setup easy.
1. Go to https://www.anaconda.com. Download the installer for your system (Windows, Mac, or Linux).
2. Run the installer. Follow the prompts.
3. Open Anaconda Navigator. Launch JupyterLab to write and run code.
4. Create a new notebook. Name it something like IAF601_Notebook.ipynb.
Anaconda includes libraries like NumPy and Pandas. These are for data work. If you have issues, check the Anaconda documentation. Repeat: Anaconda simplifies, Jupyter for interactive code.

Section 1: Variables and Assignment

Variables store data. What is a variable? It is a named box for values. Why use them? They make code reusable. How do they work? Use name = value. For example, in business, store sales.

Detailed Explanation:
What: Variable holds data like a label on a jar. Why: Easy to change values later. How: Assign with =. Common mistake: Forgetting =. Fix: Always use =. Real-world: In gene analysis, variable for sequence count. Analogy: Like naming a file. Repeat: Variables store and name data.
In [1]:
# Assign sales to a variable
sales = 1000  # This is sales in dollars
Out [1]:
No output, but sales now holds 1000.

In health analytics, variable for patient age. In cultural analytics, for text word count.

Business Insight:
In geospatial analytics, variable for latitude. Helps map points.

Section 2: Data Types

Data types define value kind. What are they? Int for numbers, float for decimals, str for text, bool for true/false. Why important? Operations depend on type. How: Python detects type.

Detailed Explanation:
What: Type tells what data is. Why: Prevents errors like adding text to number. How: Use type() to check. Mistake: Mixing types. Fix: Convert. Real-world: In biology, int for gene count. Analogy: Like food categories in fridge. Repeat: Types ensure correct operations.
In [1]:
# Examples of data types
age = 30  # int
height = 5.9  # float
name = "Alice"  # str
is_student = True  # bool
print(type(age))  # Check type
Out [1]:
<class 'int'>

In computational analytics, float for calculations. In bioinformatics, str for DNA sequences.

Section 3: Basic Operations

Operations manipulate data. What: Add, subtract, etc. Why: Compute results. How: Use +, -, *, /.

Detailed Explanation:
What: Math on variables. Why: Analyze data like profits. How: sales * 0.1 for tax. Mistake: Dividing ints gives float. Fix: Use // for integer division. Real-world: In patient analysis, add costs. Analogy: Calculator buttons. Repeat: Operations transform data.
In [1]:
# Arithmetic operations
profit = sales - 800  # Subtract costs
tax = profit * 0.05  # Multiply for tax
print(profit, tax)
Out [1]:
200 10.0

String operations: Concatenate with +. In cultural analytics, join words.

In geography, add coordinates.

Section 4: Print and Formatting

Print shows output. What: Display values. Why: See results. How: Use print().

Detailed Explanation:
What: Output to screen. Why: Debug and report. How: print("Hello", name). Mistake: Forgetting quotes for strings. Fix: Add ". Real-world: Print report in business. Analogy: Speaking out loud. Repeat: Print shares information.

Formatting: Use f-strings. f"Name: {name}".

In [1]:
# Formatting output
print(f"Patient age: {age}, Height: {height:.2f}")
Out [1]:
Patient age: 30, Height: 5.90

In health, format patient summaries. In biology, format gene stats.

Section 5: Type Conversion and Input

Convert types: int(), float(), str(). What: Change type. Why: For operations. How: new = int(old).

Detailed Explanation:
What: Switch data type. Why: To add string number. How: age = int("30"). Mistake: Converting non-number string. Fix: Check input. Real-world: Convert text data in files. Analogy: Translating languages. Repeat: Conversion makes data compatible.

Input: Get from user. value = input("Enter:")

In [1]:
# Input and conversion
user_input = input("Enter sales: ")
sales = float(user_input)
print(sales + 100)

In informatics projects, input for queries.

Key Points:
- Variables store data.
- Data types: int, float, str, bool.
- Operations: +, -, *, /.
- Print for output.
- Convert types as needed.
Repeat: Basics build foundation.
